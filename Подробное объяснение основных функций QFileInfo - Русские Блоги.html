Введение в QFileInfo

Класс QFileInfo предоставляет нам системно-независимую информацию о файле, включая имя и расположение файла в файловой системе, права доступа к файлу, каталог или ссылку и т. Д. И с помощью этого класса вы можете изменить размер файла, последнее изменение и время чтения. В то же время класс QFileInfo также можно использовать для получения информации о ресурсах Qt.

QFileInfo может представлять файлы с абсолютными или относительными путями. Среди них абсолютный путь - это путь, начинающийся с «/» (или путь, начинающийся с определенной буквы диска в системах Windows). Относительный путь - это файл или каталог относительно текущего рабочего каталога. Мы можем использовать функцию-член isRelative () этого класса, чтобы определить, представляет ли QFileInfo относительный или абсолютный путь во время разработки, или использовать makeAbsolute () для преобразования относительного пути в абсолютный путь.

Вы можете использовать функцию exists (), чтобы определить, существует ли файл, и функцию size (), чтобы получить размер файла. Функции-члены isFile (), isDir (), isSymLink () могут определять тип файла (обычный файл, каталог или символическая ссылка).

Если QFileInfo в настоящее время представляет собой символическую ссылку, функция symLinkTarget () возвращает файл, на который указывает символическая ссылка. На платформах Unix, включая macOS и iOS, символические ссылки имеют тот же размер, что и файлы, на которые они указывают, поскольку Unix прозрачно обрабатывает обычные файлы и символические ссылки; аналогично, при открытии символической ссылки для чтения и записи он фактически открывает ее. Настоящий файл указывает на. Например:

 #ifdef Q_OS_UNIX
 
  QFileInfo info1("/home/bob/bin/untabify");
  info1.isSymLink();          // returns true
  info1.absoluteFilePath();   // returns "/home/bob/bin/untabify"
  info1.size();               // returns 56201
  info1.symLinkTarget();      // returns "/opt/pretty++/bin/untabify"
 
  QFileInfo info2(info1.symLinkTarget());
  info2.isSymLink();          // returns false
  info2.absoluteFilePath();   // returns "/opt/pretty++/bin/untabify"
  info2.size();               // returns 56201
 
  #endif

На платформе Windows символические ссылки - это файлы .lnk, которые представляют собой ярлыки. В это время функция size () возвращает размер самой символической ссылки.Точно так же открытие файла .lnk означает открытие самого файла .lnk, а не реального файла, на который он указывает. Например:

#ifdef Q_OS_WIN
 
  QFileInfo info1("C:\\Documents and Settings\\Bob\\untabify.lnk");
  info1.isSymLink();          // returns true
  info1.absoluteFilePath();   // returns "C:/Documents and Settings/Bob/untabify.lnk"
  info1.size();               // returns 743
  info1.symLinkTarget();      // returns "C:/Pretty++/untabify"
 
  QFileInfo info2(info1.symLinkTarget());
  info2.isSymLink();          // returns false
  info2.absoluteFilePath();   // returns "C:/Pretty++/untabify"
  info2.size();               // returns 63942
 
  #endif

Развернуть

  1. Если QFileInfo представляет каталог, QFileInfo не будет автоматически добавлять "/" после пути. Если вам нужен этот "/", вам нужно добавить его вручную.

  2. В файловых системах NTFS, особенно Windows, права собственности на файлы и проверка разрешений по умолчанию отключены из соображений производительности. Чтобы включить эту функцию, вы можете добавить в файл следующую строку кода:

extern Q_CORE_EXPORT int qt_ntfs_permission_lookup;

  • 1

Таким образом, мы можем легко включить или отключить функцию проверки разрешений, увеличивая или уменьшая эту переменную:

qt_ntfs_permission_lookup++; // turn checking on
qt_ntfs_permission_lookup--; // turn it off again

  • 1
  • 2
  1. Некоторые функции QFileInfo запрашивают всю файловую систему, но по соображениям производительности некоторые функции управляют только самим именем файла. Например, чтобы получить абсолютный путь к файлу, absolutePath () должен запросить файловую систему. Функция path () может напрямую воздействовать на само имя файла, поэтому функция path () будет работать быстрее.
  2. Также обратите внимание, что для ускорения получения информации о файле QFileInfo кэширует информацию о файле. Но с другой стороны, файл может быть изменен другими пользователями или программами или другими функциями текущей программы в любое время, поэтому QFileInfo также предоставляет функцию refresh () для повторного чтения информации файла. Если вы хотите отключить функцию кэширования, чтобы обеспечить доступ к файловой системе для получения последней информации каждый раз, когда вы обращаетесь к информации о файле, вы можете сделать это, вызвав setCaching (false).

Имя файла и класс пути

Название функции описание
baseName Имя файла (не включает полный суффикс, не существует.Разделитель)
completeBaseName Полное имя файла (не включая последний суффикс, может существовать.Разделитель)
suffix Последний суффикс (с последнего.Начать с разделителя)
completeSuffix Полный суффикс (с первого.Начать с разделителя)
path Верните путь к файлу, не включая имя файла. Однако, если объект QFileInfo представляет путь, заканчивающийся на «/», тогда имя файла считается пустым.В это время функция возвращает полный путь.
filePath Верните имя файла, включая путь (это может быть абсолютный или относительный путь).
fileName Возвращается только имя файла, не включая путь. Обратите внимание, что если объект QFileInfo представляет путь, заканчивающийся на «/», функция возвращает пустую строку.
dir Вернуть путь к родительскому каталогу файла как объект QDir.
absoluteDir Вернуть абсолютный путь к файлу как объект QDir.
absolutePath Возвращает абсолютный путь к файлу, не включая имя файла.
absoluteFilePath Возвращает абсолютный путь к имени файла. В системах Unix будет возвращен каталог, начинающийся с «/», на платформах Windows будет возвращен каталог, начинающийся с буквы диска.
canonicalPath Верните путь к файлу (не включая имя файла), который является абсолютным путем;
canonicalFilePath Каноническое имя пути является абсолютным и уникальным. Точное определение канонического имени пути зависит от системы. При необходимости этот метод сначала преобразует имя пути в имя абсолютного пути, что имеет тот же эффект, что и вызов метода absolutefilePath (), а затем сопоставляет его с уникальным именем пути системным способом. Обычно это включает удаление избыточных элементов из имени пути (например, «.» И «...»), анализ символических ссылок (для платформ UNIX) и преобразование имен дисков в стандартный регистр (для платформ Microsoft Windows).

Дата, связанная с документом

Название функции описание
